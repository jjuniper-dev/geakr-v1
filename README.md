# geakr-v1
Here’s a more technical version:

GEAkr is a lightweight, folder-based reference architecture for scoping project-specific knowledge inputs. Each project folder acts as a bounded context containing source registries, supporting files, and optional extracted reference material. When a chatbot is directed to that folder, it uses the registered sources as the retrieval boundary, enabling responses to be grounded in project-authorized references rather than relying only on general model knowledge.

Slightly sharper / README-ready:

GEAkr is a lightweight, folder-based reference architecture for organizing project-specific knowledge inputs.
Each project folder defines a bounded context for chatbot-assisted analysis by containing a source registry, supporting documents, and optional extracted reference material. The chatbot uses the selected folder as its retrieval scope, allowing responses to be grounded in registered project references rather than general model knowledge alone.

Even more technical:

GEAkr provides a simple project-scoped retrieval pattern for chatbot-assisted work.
A GEAkr project folder functions as a bounded knowledge context. It contains source registry files, supporting documents, and optional extracted or normalized content. When selected as context, the folder constrains the chatbot’s retrieval scope so generated responses, summaries, comparisons, and artifacts are grounded in the project’s registered references.

My preferred version:

GEAkr provides a lightweight, project-scoped retrieval pattern for chatbot-assisted architecture and knowledge work. Each project folder acts as a bounded knowledge context containing registered URLs, supporting documents, and optional extracted content. When selected as context, the folder guides the chatbot toward project-authorized references, improving traceability and reducing reliance on general model knowledge alone.
