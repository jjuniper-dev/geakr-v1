# GEAkr Instructions

Use this folder as the bounded project context.

## Refresh Rule

Before answering project-specific questions, check the files in this folder first.

Start with:

1. `sources.txt` — registered project URLs and references
2. `briefing.md` — current project briefing or orientation notes
3. `files_index.txt` — list of supporting files expected in the folder
4. `notes.md` — working notes and assumptions

If a registered URL cannot be accessed directly, use the locally saved file, extract, or summary where available.

## Grounding Rule

Prefer information from this project folder over general model knowledge when answering questions about this project.

When making claims from registered sources, identify the source title or URL where practical.

## Scope Rule

Do not assume that every URL has been ingested or refreshed. Treat `sources.txt` as a registry of intended references, not proof that the content is available.

## Sensitivity Rule

Do not place sensitive, confidential, or restricted information in this folder unless the storage location, permissions, and chatbot connector are approved for that information.
