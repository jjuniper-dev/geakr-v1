# Why This Matters

GEAkr is intended to be a simple first step toward building a usable project knowledge base.

Many people begin using chatbots with unstructured prompts, scattered files, and unclear source boundaries. That can make the results feel inconsistent or hard to trust. GEAkr gives users a small, practical structure for organizing the information they want an AI assistant to consider.

## First step toward a knowledge base

GEAkr does not require a database, vector index, or formal retrieval pipeline. It starts with a folder, a few simple files, and clear instructions.

This helps users learn an important pattern:

- collect relevant sources
- keep notes close to the work
- define what is in scope
- separate source material from generated output
- ask the AI assistant to work from a bounded context

That is the foundation of a more mature knowledge base.

## Practical AI literacy

GEAkr can also help users understand that different AI tools behave differently. When the same folder, sources, and prompt are used across more than one assistant, users can compare the results and observe differences in:

- answer quality
- source use
- reasoning
- summarization
- instruction following
- artifact generation
- handling of uncertainty

This can help users move from abstract opinions about AI to direct experience with AI capabilities and limitations.

## Adoption principle

Start small. Use non-sensitive material. Try the pattern with a simple project. Keep what works. Improve the folder as the work becomes clearer.

The goal is not to force a tool or platform. The goal is to help users see the value of organizing knowledge so AI assistants can work with it more effectively.
