# How GEAkr Helps Prevent Shadow AI

Date: 2026-04-25
Status: v0.1 positioning

## Core position

GEAkr helps prevent shadow AI by giving users a simple, approved-feeling lane for AI-assisted work instead of leaving them to improvise with scattered prompts, unmanaged files, and untraceable outputs.

It does not prevent shadow AI by blocking experimentation. It reduces shadow AI by making the governed path easier, clearer, and more useful.

---

## Shadow AI problem

Shadow AI emerges when users need AI help but do not have:

- clear guidance on what information can be used
- a simple way to structure context
- approved tools that meet their workflow needs
- repeatable prompts or patterns
- traceable outputs
- confidence that governance is practical rather than obstructive

When official paths are too slow, vague, or cumbersome, users create their own informal workflows.

---

## GEAkr response

GEAkr provides a lightweight operating pattern:

```text
Sources + notes + files + instructions
        ↓
Bounded project context
        ↓
AI-assisted output
        ↓
Human review and governance handoff
```

This gives users a practical alternative to unmanaged AI use.

---

## How GEAkr reduces shadow AI risk

### 1. Makes the safe path easy

Users can start with a simple folder and a few files instead of a complex governance process.

### 2. Defines context boundaries

GEAkr separates:
- public sources
- internal unclassified sources
- personal/PCA sources
- promoted/shared knowledge

This helps prevent accidental mixing of contexts.

### 3. Captures source traceability

A structured `sources.txt`, knowledge files, and graph links make it easier to know where outputs came from.

### 4. Encourages approved-tool use

GEAkr is tool-agnostic, but it can be deployed through approved environments for work use.

The pattern can support:
- approved chat tools
- approved Azure runtime
- approved GitHub/SharePoint workflows
- approved internal hosting

### 5. Keeps outputs reviewable

GEAkr separates extracted knowledge from synthesized responses. Immediate PCA-style responses are provisional, not authoritative.

### 6. Enables governance escalation

Simple use can later escalate into:
- GitHub version control
- pull request review
- CI/CD validation
- metadata checks
- AIA/Directive support artifacts
- architecture decision support

---

## Important caveat

GEAkr is not itself an approval mechanism, security boundary, or compliance control.

GEAkr supports governance by improving structure, traceability, reviewability, and repeatability.

The runtime, storage, connectors, and data classification still determine whether a specific use is appropriate.

---

## Recommended executive framing

GEAkr provides a low-friction governed lane for AI-assisted work. It reduces shadow AI risk by giving teams a simple, repeatable way to organize context, use approved tools, preserve traceability, and produce reviewable outputs.

GEAkr helps prevent shadow AI by making the governed path easier to use than the unmanaged path.
